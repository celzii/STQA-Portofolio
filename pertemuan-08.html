Pertemuan-08 — Pengantar Cypress

Automasi pengujian web modern berbasis JavaScript

1. Definisi Cypress

Cypress adalah framework pengujian end-to-end berbasis JavaScript yang digunakan untuk menguji aplikasi web langsung di browser. Cypress memungkinkan pengembang melihat secara real-time setiap langkah pengujian, dari membuka halaman, mengisi form, hingga memverifikasi hasil.

Berbeda dari Selenium, Cypress bekerja langsung di dalam browser (run-time environment) sehingga hasilnya lebih cepat dan stabil.

2. Keunggulan Cypress

  • Arsitektur modern: Cypress berjalan langsung di browser, bukan melalui driver eksternal seperti Selenium.
  • Test Runner interaktif: Menampilkan setiap langkah pengujian dan hasilnya secara visual.
  • Time Travel: Fitur unik yang memungkinkan pengguna melihat kembali setiap langkah pengujian dengan screenshot otomatis.
  • Realtime reload: Tes diperbarui secara otomatis saat kode diubah.
  • Integrasi CI/CD mudah: Cypress mendukung pipeline otomatisasi seperti GitHub Actions, Jenkins, atau GitLab CI.

3. Instalasi Cypress via NPM/Yarn

Cypress dapat diinstal melalui NPM atau Yarn. Pastikan Node.js sudah terpasang terlebih dahulu. 


# Instalasi menggunakan NPM
npm install cypress --save-dev

# Atau menggunakan Yarn
yarn add cypress --dev

Setelah instalasi berhasil, jalankan perintah berikut untuk membuka Test Runner: 

npx cypress open

4. Struktur Tes di Cypress

Struktur file pengujian Cypress biasanya menggunakan pola describe–it yang mirip dengan framework testing lain. 


// contoh_test.cy.js
describe('Judul Skenario Pengujian', () => {
  it('Langkah uji pertama', () => {
    // kode pengujian di sini
  })
})
  • describe() → digunakan untuk mengelompokkan skenario uji.
  • it() → mendefinisikan satu kasus uji (test case).

5. Perintah Dasar Cypress

Perintah Dasar Cypress
Perintah Deskripsi Contoh
cy.visit() Membuka URL tertentu. cy.visit('https://www.saucedemo.com/')
cy.get() Mengambil elemen HTML berdasarkan selector. cy.get('#user-name')
cy.type() Mengetik teks ke dalam input field. cy.get('#password').type('secret_sauce')
cy.click() Mengklik elemen seperti tombol atau link. cy.get('#login-button').click()
cy.contains() Mencari elemen yang mengandung teks tertentu. cy.contains('Products')

6. Studi Kasus: Login ke SauceDemo.com dengan Cypress

Berikut contoh skrip Cypress untuk menguji fitur login di situs SauceDemo. 


// cypress/e2e/login_saucedemo.cy.js
describe('Testing Login SauceDemo', () => {
  it('Login berhasil dengan kredensial valid', () => {
    cy.visit('https://www.saucedemo.com/')

    // Mengisi username dan password
    cy.get('#user-name').type('standard_user')
    cy.get('#password').type('secret_sauce')

    // Klik tombol login
    cy.get('#login-button').click()

    // Verifikasi halaman berhasil masuk
    cy.url().should('include', '/inventory')
    cy.contains('Products').should('be.visible')
  })

  it('Login gagal dengan password salah', () => {
    cy.visit('https://www.saucedemo.com/')
    cy.get('#user-name').type('standard_user')
    cy.get('#password').type('wrong_password')
    cy.get('#login-button').click()

    // Verifikasi pesan error muncul
    cy.get('[data-test="error"]').should('contain', 'Epic sadface')
  })
})

7. Test Scenario dan Test Case — SauceDemo.com

Test Scenario

  • Scenario: Memverifikasi proses login pada situs SauceDemo.
  • Precondition: Browser terbuka dan halaman login aktif.
  • Expected Result: Pengguna dapat login menggunakan kredensial valid dan ditolak jika salah.

8. Kesimpulan

Cypress menghadirkan cara baru dalam melakukan pengujian otomatis berbasis web dengan pendekatan yang cepat, stabil, dan interaktif. Dengan fitur visual test runner dan integrasi langsung di browser, Cypress menjadi pilihan utama untuk pengujian modern terutama pada aplikasi berbasis JavaScript.

Contoh pengujian di SauceDemo memperlihatkan betapa mudahnya melakukan uji login menggunakan perintah sederhana seperti cy.get(), cy.type(), dan cy.click().
