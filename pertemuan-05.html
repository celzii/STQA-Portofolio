Pertemuan-05 — Pengantar Unit Testing

Menguji komponen kecil untuk menjamin kualitas besar

1. Definisi Unit Testing

Unit Testing adalah proses pengujian terhadap bagian terkecil dari kode program, biasanya berupa fungsi, metode, atau modul, untuk memastikan bahwa setiap unit bekerja dengan benar sesuai dengan desain dan kebutuhan yang ditetapkan.

Tujuan utama unit testing adalah mendeteksi kesalahan sejak dini dalam tahap pengembangan sebelum kode digabungkan ke dalam sistem yang lebih besar.

2. Keunggulan Unit Testing

  • Deteksi dini bug: Kesalahan ditemukan lebih cepat sebelum menjadi masalah besar.
  • Mempermudah debugging: Unit testing membantu mengisolasi area yang menyebabkan error.
  • Meningkatkan kualitas kode: Pengembang terdorong untuk menulis kode yang modular dan mudah diuji.
  • Mendukung perubahan (refactoring): Pengujian otomatis memastikan fungsi lama tetap berjalan saat kode diperbarui.
  • Meningkatkan kepercayaan terhadap sistem: Setiap bagian telah diverifikasi secara individual sebelum digabungkan.

3. Framework & Tools untuk Unit Testing

Berbagai bahasa pemrograman memiliki framework yang memudahkan proses unit testing. Berikut adalah beberapa yang paling populer:

Framework Unit Testing Populer
Bahasa Pemrograman Framework Unit Test Keterangan

4. Struktur Tes: Pola Arrange – Act – Assert (AAA)

Struktur paling umum dalam menulis unit test adalah pola AAA:

  • Arrange: Menyiapkan kondisi awal dan data yang dibutuhkan untuk pengujian.
  • Act: Menjalankan fungsi atau kode yang akan diuji.
  • Assert: Memverifikasi bahwa hasil aktual sesuai dengan hasil yang diharapkan.

# Contoh pola AAA dalam Pytest
def test_penjumlahan():
    # Arrange
    a = 5
    b = 3

    # Act
    hasil = a + b

    # Assert
    assert hasil == 8

5. Assertions

Assertion adalah pernyataan yang digunakan dalam pengujian untuk memverifikasi hasil aktual dengan hasil yang diharapkan. Jika pernyataan ini salah, maka pengujian dianggap gagal.

Contoh Assertions Umum (Pytest)

  • assert hasil == nilai_ekspektasi → memeriksa kesamaan nilai.
  • assert x in daftar → memastikan elemen ada dalam koleksi.
  • assert isinstance(obj, ClassName) → memastikan tipe data benar.
  • assert not kondisi → memastikan kondisi bernilai salah.

6. Studi Kasus: Unit Test Aplikasi BMI (Python & Pytest)

Misalkan terdapat fungsi sederhana untuk menghitung nilai BMI berdasarkan berat dan tinggi badan. Berikut contoh implementasi dan unit test menggunakan Pytest: 


# bmi.py
def hitung_bmi(berat, tinggi):
    if tinggi <= 0:
        raise ValueError("Tinggi harus lebih dari 0")
    bmi = berat / ((tinggi / 100) ** 2)
    return round(bmi, 2)

# test_bmi.py
from bmi import hitung_bmi

def test_bmi_normal():
    # Arrange
    berat = 60
    tinggi = 165

    # Act
    hasil = hitung_bmi(berat, tinggi)

    # Assert
    assert hasil == 22.04

def test_bmi_invalid_tinggi():
    # Arrange
    berat = 60
    tinggi = 0

    # Act & Assert
    try:
        hitung_bmi(berat, tinggi)
    except ValueError as e:
        assert str(e) == "Tinggi harus lebih dari 0"

Hasil pengujian akan menampilkan dua status: Passed (uji berhasil) atau Failed (uji gagal), tergantung dari kesesuaian hasil aktual dengan ekspektasi. Pytest dapat dijalankan dengan perintah: 

pytest test_bmi.py

7. Kesimpulan

Unit Testing merupakan tahap penting untuk menjamin bahwa setiap bagian kecil dari sistem bekerja dengan benar. Dengan framework seperti Pytest, JUnit, atau Jest, pengujian dapat dilakukan secara otomatis, cepat, dan efisien. Pola Arrange–Act–Assert serta penggunaan assertion membantu menjaga struktur pengujian tetap jelas dan mudah dipelihara dalam proyek perangkat lunak berskala besar.
