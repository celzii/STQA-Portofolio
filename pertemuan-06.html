Pertemuan-06 — Pengantar API Testing

Memastikan integrasi antar sistem berjalan mulus melalui pengujian API

1. Definisi API Testing

API (Application Programming Interface) Testing adalah proses pengujian untuk memverifikasi apakah antarmuka pemrograman aplikasi (API) bekerja sesuai spesifikasi. Pengujian dilakukan untuk memastikan bahwa komunikasi antara server dan client berjalan dengan benar, data ditransfer dengan akurat, dan respon yang diberikan sesuai harapan.

Tidak seperti UI testing yang berfokus pada tampilan, API testing menguji logika bisnis, fungsionalitas, keamanan, dan performa dari layer backend sistem.

2. Keunggulan API Testing

  • Lebih cepat dan efisien: Pengujian API dapat dilakukan tanpa melalui antarmuka pengguna (UI).
  • Deteksi bug lebih awal: Kesalahan logika bisnis dapat ditemukan sebelum tahap integrasi UI.
  • Stabil dan mudah diotomasi: API jarang berubah, sehingga cocok untuk pengujian otomatis (automation testing).
  • Mendukung integrasi sistem: API testing memastikan koneksi antar aplikasi pihak ketiga berjalan baik.
  • Meningkatkan keandalan layanan: Memastikan sistem backend memberikan respon yang benar dan konsisten.

3. Tools untuk API Testing

Beberapa alat yang umum digunakan untuk melakukan pengujian API antara lain:

  • Postman: Alat populer dengan antarmuka grafis untuk mengirim permintaan (request) dan memeriksa respon API.
  • SoapUI: Tool pengujian berbasis XML yang mendukung protokol SOAP dan REST.
  • cURL: Alat berbasis command line untuk mengirim request HTTP secara manual.
  • Insomnia: Alternatif Postman dengan antarmuka sederhana dan dukungan berbagai metode HTTP.

Postman sebagai Tool Utama

Postman memungkinkan pengembang atau penguji untuk membuat, mengirim, dan memverifikasi request API dengan mudah. Fitur utamanya meliputi:

  • Antarmuka visual untuk membuat permintaan GET, POST, PUT, dan DELETE.
  • Dukungan environment variable (untuk URL, token, dan parameter dinamis).
  • Collection runner untuk eksekusi batch testing.
  • Integrasi otomatisasi dan scripting (Pre-request Script & Test Script).

4. Konsep Dasar: Anatomi Request & Response API

Dalam pengujian API, penting untuk memahami komponen dasar dari komunikasi HTTP yang melibatkan Request dan Response.

Bagian dari Request

  • URL / Endpoint: Alamat API yang diakses, contoh: https://api.example.com/users
  • Method: Jenis operasi yang dilakukan, seperti:
    • GET – Mengambil data
    • POST – Mengirim data baru
    • PUT – Memperbarui data
    • DELETE – Menghapus data
  • Headers: Informasi tambahan (contoh: format konten, token otentikasi).
  • Body: Data yang dikirim bersama request (biasanya dalam format JSON).

Bagian dari Response

  • Status Code: Menunjukkan hasil dari request, contoh:
    • 200 OK – Permintaan berhasil.
    • 201 Created – Data berhasil dibuat.
    • 400 Bad Request – Format permintaan salah.
    • 404 Not Found – Endpoint tidak ditemukan.
    • 500 Internal Server Error – Kesalahan pada server.
  • Response Body: Isi balasan dari server (biasanya dalam format JSON).
  • Headers: Metadata tambahan seperti tipe konten, waktu respon, dan server.

5. Operasi Dasar Menggunakan Postman

  1. Buka Postman dan buat Request baru.
  2. Pilih metode HTTP (misalnya GET atau POST).
  3. Masukkan URL endpoint API, misalnya: https://jsonplaceholder.typicode.com/posts.
  4. Jika menggunakan POST, pilih tab Body dan isi data dalam format JSON.
  5. Klik Send untuk mengirim request dan lihat hasilnya pada tab Response.

Contoh Request GET


GET https://jsonplaceholder.typicode.com/posts/1

Contoh Request POST


POST https://jsonplaceholder.typicode.com/posts
Content-Type: application/json

{
  "title": "Belajar API Testing",
  "body": "API Testing membantu memastikan komunikasi antar sistem berjalan baik.",
  "userId": 1
}

Contoh Response (JSON)


{
  "id": 101,
  "title": "Belajar API Testing",
  "body": "API Testing membantu memastikan komunikasi antar sistem berjalan baik.",
  "userId": 1
}

6. Studi Kasus: Live Coding API Testing (GET & POST)

Studi kasus berikut menggunakan API publik https://jsonplaceholder.typicode.com. API ini menyediakan data dummy yang dapat digunakan untuk latihan pengujian.

Langkah Pengujian GET

  1. Buka Postman.
  2. Buat request GET ke endpoint /users.
  3. Klik Send.
  4. Perhatikan bahwa respon berupa daftar pengguna dalam format JSON.

Langkah Pengujian POST

  1. Buat request baru dengan metode POST.
  2. Gunakan endpoint /posts.
  3. Isi Body dengan format JSON, misalnya:

{
  "title": "Coba API POST",
  "body": "Ini adalah contoh pengujian POST di Postman",
  "userId": 2
}

Setelah menekan tombol Send, server akan mengembalikan respon dengan kode 201 Created dan JSON berisi data baru yang berhasil dibuat.

7. Kesimpulan

API Testing memastikan bahwa komunikasi antar sistem bekerja dengan benar, cepat, dan aman. Dengan menggunakan alat seperti Postman atau SoapUI, penguji dapat memverifikasi setiap endpoint API, mengukur kinerja, serta mengotomatiskan proses pengujian untuk integrasi yang andal.
